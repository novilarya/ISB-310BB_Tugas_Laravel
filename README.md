Pada minggu ke-5, pengembangan proyek difokuskan pada penguatan arsitektur **MVC (Model-View-Controller)** melalui integrasi basis data yang lebih dinamis. Fokus utamanya adalah penerapan relasi antar tabel (*Eloquent Relationships*), otomatisasi skema database melalui *Migrations*, serta manajemen data contoh menggunakan *Seeders* untuk memastikan aplikasi dapat menyajikan informasi produk secara terstruktur.
### 1. Perancangan Skema Database (Migrations)
Otomatisasi struktur tabel database dikelola melalui file migrasi untuk tabel `categories`, `brands`, dan `products`. Tabel `products` dirancang secara kompleks dengan menyertakan *Foreign Key* `category_id` dan `brand_id` yang menghubungkannya ke tabel induk masing-masing. Penggunaan fitur `onDelete('cascade')` pada migrasi ini sangat krusial karena menjamin integritas data. Jika data kategori atau brand dihapus, maka seluruh produk yang berkaitan akan ikut terhapus secara otomatis di database.
### 2. Implementasi Model & Eloquent Relationships
Logika data didefinisikan pada folder `app/Models/` untuk mengatur perilaku entitas di dalam aplikasi. Model `category` dan `brand` menerapkan relasi *One-to-Many* menggunakan fungsi `hasMany`, yang menandakan bahwa satu kategori atau brand dapat membawahi banyak produk sekaligus. Sebaliknya, model `product` bertindak sebagai entitas utama yang menggunakan relasi *inverse* `belongsTo`, sehingga data merek dan kategori terkait dapat diakses langsung melalui objek produk.
### 3. Manajemen Data Dummy (Seeding)
Untuk mempermudah proses pengujian antarmuka oleh pengembang yang lain, digunakan fitur *Seeder* pada file `DatabaseSeeder.php`. Di dalam fungsi `run()`, sistem menjalankan perintah `insert()` secara berurutan untuk mengisi tabel kategori, brand, dan terakhir tabel produk dengan data contoh secara otomatis. Hal ini sangat membantu tahap pengembangan agar pengembang tidak perlu mengisi data secara manual melalui database langsung untuk menguji render tampilan produk pada layar browser.
### 4. Pengaturan Controller & Eager Loading
Pada `ProductController.php` berguna untuk menghubungkan Model dengan View. Di dalam fungsi `index()`, sistem mengambil seluruh data kategori dan brand, serta mengambil data produk menggunakan teknik *eager loading* melalui perintah `with(['category', 'brand'])`. Teknik ini secara signifikan meningkatkan efisiensi performa karena Laravel hanya menjalankan satu query untuk mengambil data produk beserta seluruh relasinya sebelum dikirimkan ke tampilan menggunakan fungsi `compact`.
### 5. Presentasi Antarmuka Dinamis (Blade View)
Penyajian data kepada pengguna dikelola melalui file `product.blade.php` yang memanfaatkan kekuatan *Blade Templating Engine*. Data produk yang diterima dari controller ditampilkan menggunakan `@foreach` ke dalam komponen kartu (*card*) yang rapi, termasuk menampilkan lencana (*badge*) merek dan kategori melalui akses relasi model. Selain itu, harga produk diformat menggunakan *helper* `number_format` untuk memastikan informasi ditampilkan sesuai standar mata uang Rupiah yang mudah dipahami oleh pengguna.
### 6. Konfigurasi Jalur Akses (Routing)
Seluruh titik akses URL aplikasi didefinisikan pada file `routes/web.php`. Jalur `/products` didaftarkan menggunakan metode `GET` yang diarahkan langsung ke fungsi `index` pada `ProductController`. Penggunaan *named routes* seperti `->name('products')` memberikan kemudahan dalam pemanggilan URL di seluruh bagian aplikasi, baik pada elemen navigasi di `index.blade.php` maupun pada logika pengalihan di controller, tanpa perlu menuliskan jalur URL secara manual yang rawan kesalahan.
